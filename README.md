# WebAR Art Gallery 🖼️✨

Веб-приложение дополненной реальности (AR), которое "оживляет" картины прямо в браузере мобильного телефона. Без необходимости скачивать сторонние приложения — просто откройте ссылку, наведите камеру на изображение, и поверх него запустится подготовленное видео.

## 🌟 Особенности проекта
* **Работает прямо в браузере (WebAR):** Пользователям нужен только смартфон и доступ в интернет.
* **Кроссплатформенность:** Стабильно работает как на Android (Chrome), так и на iOS (Safari).
* **Multi-target распознавание:** Приложение способно хранить в памяти и распознавать сразу несколько картин в рамках одной сцены.

## ⚙️ Как это работает (How It Works)

Ниже подробно описан полный цикл работы приложения — от открытия страницы до воспроизведения видео поверх картины.

### Этап 1: Загрузка страницы

Когда пользователь открывает страницу в браузере, происходит следующее:
1. Загружаются две основные библиотеки: **A-Frame** (3D-движок для браузера) и **MindAR** (библиотека распознавания изображений).
2. Загружается файл **`targets.mind`** (~8 МБ) — это скомпилированная база изображений-маркеров (картин), по которой MindAR будет выполнять распознавание.
3. **Видеофайлы НЕ загружаются** на этом этапе. В HTML-коде элементы `<video>` объявлены без атрибута `src` — это принцип **lazy loading** (ленивая загрузка). Благодаря этому страница загружается за несколько секунд, а не ждёт скачивания ~229 МБ видео.
4. Пользователь видит кнопку **«Batyrmany basynyz (Start AR)»**.

### Этап 2: Нажатие кнопки «Start AR» (разблокировка видео)

Мобильные браузеры (особенно Safari на iOS) **блокируют автоматическое воспроизведение видео** по соображениям безопасности. Видео можно проиграть программно только если до этого пользователь совершил жест (tap, click). Поэтому нажатие кнопки — это не просто UI, а **критически важный момент**:

1. **Разблокировка AudioContext:** Создаётся и запускается `AudioContext` — Web API для работы с аудио. Это снимает блокировку звука на странице.
2. **Разблокировка видеоэлементов:** Каждому из 10 `<video>` элементов подставляется крошечный беззвучный видеофайл (закодирован как data URI прямо в коде, ~1 КБ). Он проигрывается в `muted` режиме и тут же ставится на паузу. После этого браузер «запоминает», что пользователь разрешил воспроизведение, и дальнейшие вызовы `video.play()` из JavaScript будут работать без ошибок.
3. **Запуск AR-движка:** Вызывается `sceneEl.systems["mindar-image-system"].start()`, который:
   - Запрашивает доступ к **камере** устройства.
   - Начинает захват видеопотока с камеры.
   - Запускает покадровый анализ изображения через алгоритмы MindAR.

### Этап 3: Распознавание изображения (Image Tracking)

После запуска камеры MindAR непрерывно анализирует каждый кадр видеопотока:

1. **Извлечение ключевых точек (feature points):** Из каждого кадра камеры алгоритм извлекает характерные визуальные признаки (углы, контрасты, текстуры).
2. **Сопоставление с базой:** Эти признаки сравниваются с заранее скомпилированными признаками из файла `targets.mind`. Каждое изображение в этом файле имеет свой `targetIndex` (0, 1, 2, ... 9).
3. **Событие `targetFound`:** Когда совпадение найдено с достаточной уверенностью, MindAR генерирует событие `targetFound` на соответствующем `<a-entity>` элементе. В нём же содержится информация о позиции и ориентации маркера в пространстве — так 3D-объект (видео) корректно «привязывается» к картине.
4. **Событие `targetLost`:** Когда картина выходит из поля зрения камеры — генерируется `targetLost`.

### Этап 4: Lazy-загрузка и воспроизведение видео

Когда срабатывает `targetFound`, запускается функция `loadAndPlay`:

1. **Проверка кеша:** Если видео для этого таргета уже загружалось ранее (хранится в `Set` объекте `loadedVideos`), оно просто перематывается на начало и воспроизводится **мгновенно** — без задержки.
2. **Первая загрузка:** Если видео загружается впервые:
   - На экране появляется **спиннер загрузки** (CSS-анимация вращающегося кольца).
   - Атрибуту `src` элемента `<video>` присваивается URL файла (например, `./resources/video3.mp4`).
   - Вызывается `video.load()` — браузер начинает скачивать видеофайл.
   - Скрипт подписывается на событие `canplay` — оно срабатывает, когда браузер скачал достаточно данных для начала воспроизведения (обычно первые несколько секунд).
   - После `canplay` — спиннер скрывается и видео воспроизводится.
   - **Защитный таймаут (8 сек):** Если `canplay` не сработал за 8 секунд (слабый интернет), спиннер скрывается и делается попытка проигрывания.
3. **Управление конфликтами:** Перед запуском нового видео все остальные видео ставятся на паузу (`pauseAllExcept`). Мобильные браузеры часто позволяют проигрывать только один видеопоток одновременно.

### Этап 5: Видео на экране (Рендеринг в AR)

1. A-Frame использует элемент `<video>` как **текстуру** для 3D-плоскости (`<a-video>`).
2. Эта плоскость прикреплена к AR-маркеру (`<a-entity mindar-image-target>`), поэтому она двигается и поворачивается вместе с картиной в реальном мире.
3. Когда пользователь уводит камеру от картины → `targetLost` → видео ставится на паузу → плоскость скрывается.

### Схема процесса

```
Пользователь открывает страницу
         │
         ▼
  Загрузка A-Frame + MindAR + targets.mind
  (видео НЕ загружаются)
         │
         ▼
  Кнопка "Start AR"
         │
         ▼
  ┌──────────────────────────────┐
  │  1. Разблокировка AudioContext │
  │  2. Разблокировка <video>      │
  │  3. Запуск камеры + MindAR     │
  └──────────────────────────────┘
         │
         ▼
  Покадровый анализ видеопотока камеры
         │
    ┌────┴────┐
    │         │
targetFound  (ожидание)
    │
    ▼
  ┌────────────────────────┐
  │  Видео уже загружено?  │
  │                        │
  │  Да → play() мгновенно │
  │  Нет → set src →       │
  │    показать спиннер →  │
  │    ждать canplay →     │
  │    play()              │
  └────────────────────────┘
    │
    ▼
  Видео отображается поверх картины как AR-текстура
    │
    ▼
  targetLost → pause() → скрытие
```

## 🛠 Стек технологий
* **[A-Frame](https://aframe.io/):** Web-фреймворк для создания 3D и AR/VR веб-приложений.
* **[MindAR](https://hiukim.github.io/mind-ar-js-doc/):** Библиотека отслеживания изображений (Image Tracking) для веба.
* **HTML5 / Vanilla JavaScript**
* **Хостинг:** GitHub Pages.

## 🚀 Как запустить проект локально
Для работы камеры и AR-движка в браузере требуется защищенное соединение (HTTPS) или локальный сервер. Открыть файл `index.html` двойным кликом не получится.

1. Склонируйте репозиторий.
2. Откройте проект в редакторе кода (например, VS Code).
3. Запустите проект через расширение **Live Server**.
4. Убедитесь, что ваш телефон и компьютер находятся в одной Wi-Fi сети, и откройте локальный IP-адрес (например, `http://192.168.1.X:5500`) на смартфоне.

## 📝 Заметка для разработчика: Подготовка медиафайлов

### Маркеры (Изображения для распознавания)
Все картины-триггеры должны быть скомпилированы в один общий файл `.mind`. Для добавления новых картин:
1. Зайдите в [MindAR Compiler](https://hiukim.github.io/mind-ar-js-doc/tools/compile/).
2. Загрузите нужные изображения (порядок загрузки определяет `targetIndex` в коде: 0, 1, 2 и т.д.).
3. Скачайте файл `targets.mind` и замените им старый в корне проекта.

### Требования к видео (Критично для iOS)
Мобильный браузер Safari (iOS) крайне строго относится к воспроизведению видео в WebGL. Чтобы видео стабильно работало на iPhone, оно должно соответствовать следующим требованиям:
* **Кодек:** Строго **H.264** (H.265/HEVC отображаться не будут).
* **Атрибуты в HTML:** Видео должно иметь следующие теги для обхода блокировок автовоспроизведения: `autoplay loop muted playsinline webkit-playsinline`.
* Видео запускается только после явного взаимодействия пользователя со страницей (через кнопку "Начать").
